# CodeMate

CodeMate is a competitive coding arena designed for students, teachers, and admins. The project combines a C++ console-based backend with a lightweight browser UI to manage users, questions, races, leaderboards, and teacher/admin workflows.

## Project Overview

This repository is structured as a small full-stack-style prototype:

- The backend logic lives in `main.cpp`
- User, question, and race data are persisted in plain text files
- The frontend is a static web app in `frontend/`
- The C++ app exports JSON snapshots into `frontend/data/` so the browser can visualize the state

## Main Features

- Student registration and login
- Teacher role with topic-based question creation
- Admin approval flow for teachers
- Competitive race gameplay with topic-based matching
- Leaderboard and profile-style dashboard
- Skill tiering, badges, streaks, and scoreboard tracking
- Browser GUI for dashboard, race room, teacher hub, and admin panel

## Technology Stack

- Language: C++17
- Frontend: HTML, CSS, JavaScript
- Data storage: flat text files (`users.txt`, `questions.txt`, `races.txt`)
- Frontend data export: JSON snapshots under `frontend/data/`

## Repository Structure

```text
CodeMate/

│   ├── index.html           # Main web interface

│       ├── users.json       # Generated user snapshot for frontend
│       └── questions.json   # Generated question snapshot for frontend

```

## How It Works

1. Run the C++ application.
2. Register a user or log in.
3. Use the built-in menu to access dashboard, race, teacher, and admin flows.
4. Choose option `3` in the console app to launch the web GUI.
5. The frontend reads the generated JSON data and presents it in a browser.

## Build and Run

Compile the C++ backend from the project root:

```bash
g++ main.cpp -std=c++17 -O2 -o codemate.exe
```

Run the application on Windows:

```bash
codemate.exe
```

Or, if you want to start the browser interface directly, open:

```text
frontend/index.html
```

> Note: the web UI depends on `frontend/data/users.json` and `frontend/data/questions.json`, which are generated by the C++ app when it runs.

## Data Files

The project stores its main state in these files:

- `users.txt` – user profiles, stats, roles, and permissions
- `questions.txt` – question bank and metadata
- `races.txt` – race results history
- `frontend/data/users.json` – frontend-friendly user snapshot
- `frontend/data/questions.json` – frontend-friendly question snapshot

## Notes

- The application is designed as a lightweight prototype rather than a production web backend.
- Data persistence is file-based and intentionally simple.
- The frontend is static and client-side, so it is best suited for local demos and educational use.
